# ronpicard.com

This repository is the **primary** home for my personal website. The site is a static React app (Vite) deployed to **GitHub Pages**. I am pointing **ronpicard.com** at this deployment (custom domain on the `gh-pages` branch), so visitors to the domain will see this build rather than the previous Squarespace-hosted site.

Post content is driven by `src/data/siteArticles.json`, which is generated by scraping the legacy Squarespace URLs (sitemap + post HTML) until that source is retired. The home page lists every post from that file; article pages show **Demo** / **Code** only when the original post had matching buttons, embedded **GitHub Pages** demos, **YouTube** and other iframe embeds when detected, mirrored **images and static assets** under `public/resources/`, summaries and body HTML where available, and prev/next navigation (newest first).

## Refreshing content after publishing

While the legacy Squarespace site is still the scrape source (until it is fully retired):

```bash
npm run sync:articles
npm run mirror:resources
```

Then commit `src/data/siteArticles.json`, `public/resources/`, and `scripts/resource-manifest.json` as needed.

`npm run build:full` runs TypeScript, the mirror step, and Vite (used before deploy). `npm run build` skips the mirror and assumes assets are already present.

Routes use **hash URLs** (for example `https://ronpicard.github.io/ronpicard.com/#/blog/sorting-algorithm-visualizer-web-app`) so client-side routing works on GitHub Pages without extra server configuration.

## Commands

- `npm install` — install dependencies
- `npm run dev` — local dev server
- `npm run build` — production build to `dist/` (no network; uses committed JSON + `public/resources/`)
- `npm run build:full` — `tsc`, mirror remote assets, then Vite build
- `npm run mirror:resources` — download images/PDFs/ZIP referenced in JSON into `public/resources/` and rewrite local paths
- `npm run deploy` — `build:full`, then publish `dist/` to the `gh-pages` branch (via [gh-pages](https://github.com/tschaub/gh-pages))
- `npm run sync:articles` — re-fetch `https://www.ronpicard.com/sitemap.xml` and each post; overwrite `src/data/siteArticles.json`

## GitHub Pages and custom domain

1. Repo **Settings → Pages**: set **Source** to **Deploy from a branch**, branch **gh-pages**, folder **/ (root)**.
2. Default GitHub URL: `https://ronpicard.github.io/ronpicard.com/` (matches `base` in `vite.config.ts` and `homepage` in `package.json`).

When **ronpicard.com** uses this repo’s GitHub Pages custom domain, set `base` in `vite.config.ts` to `'/'` and update `homepage` in `package.json` to `https://ronpicard.com/` (or your canonical URL). Add the domain under **Pages → Custom domain** and configure DNS as GitHub documents.

## Security notes

The app is **fully static** (HTML/CSS/JS only): no server, database, or API in this repo, which keeps the attack surface small. External links use `rel="noopener noreferrer"`. Keep dependencies updated (`npm audit`) before deploys.
